Quant Research Stack

From raw data to a shareable research dashboard.

This project is not just a factor notebook. It validates data, builds portfolios, simulates execution, reconciles fills, and exports an HTML report someone else can evaluate in minutes.

What it already does

Validation, manifests, optimizer-based construction, shorting assumptions, capacity analysis, execution reconciliation, factor diagnostics, stress reporting, and order blotters are already wired into the CLI.

Bundle command

Publish target

The command copies the latest dashboard and diagnostics into docs/demo/ and generates a static entry page at docs/demo/index.html.

Strategy gallery

Gallery command

python run_quant.py gallery --config config/sample_config.json

What it adds

The gallery runs multiple presets, publishes a bundle for each one, and creates docs/demo/gallery.html with sorting, tag filtering, visible preset override diffs, per-preset share cards, a root demo spotlight, and winner artifacts for badges, README snippets, release notes, and social posts.

Why it is more credible than a toy backtest

Validation layer

Coverage, linkage, missingness, and rebalance-level survivorship are exported before performance is trusted.

Execution realism

Participation-aware costs, multi-day execution, backlog behavior, and fill reconciliation are built in.

Shareable outputs

Signals, orders, diagnostics, manifests, and a static dashboard are all written to files instead of hidden in notebook state.

What still blocks breakout adoption

Hosted demo is sample-only

A GitHub Pages path now exists, but it still publishes one curated sample bundle rather than user-generated runs.

No strategy gallery

People need prebuilt examples and visual comparisons, not just a config file and CSV outputs.

No social loop

There is no public scoreboard, sharable result URL, or collaborative layer yet.
